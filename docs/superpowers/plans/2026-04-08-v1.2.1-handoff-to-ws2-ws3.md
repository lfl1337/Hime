## Two latent v1.2.0 bugs that shipped with v1.2.1 WS1+WS4

These are genuinely important — future sessions should be aware.

### Bug 1: NameError in `routers/training.py` (AUDIT gap, fixed by WS1 T10)

**Discovery:** WS1 T10 (API hyperparam bounds) found that the existing `routers/training.py` at commit `f5fe378` (main's HEAD pre-v1.2.1) was missing `Field` from its pydantic import:

```python
from pydantic import BaseModel, field_validator  # BROKEN
```

But the class `StartTrainingRequest` referenced `Field(...)` several times. Importing the module would have raised `NameError` at class-definition time, breaking FastAPI startup via `app.include_router(training.router)`.

**Why nobody noticed:** The user's local **uncommitted** fix (`+Field` in the import line) masked the bug at runtime. Git status at the start of this session showed `M app/backend/app/routers/training.py`.

**Fix:** WS1 T10 commit `043ee19` added the same `+Field` line. On merge, the user's local fix merged cleanly (same line). Stashed-and-popped during merge — stash was a no-op after merge.

**Action items for next session:** None for this specific bug. Just be aware that if you see similar latent NameErrors in other files, check whether the main worktree has uncommitted fixes masking them.

### Bug 2: NameError in `epub_service._validate_epub_path` (AUDIT gap, fixed by WS4 T2)

**Discovery:** WS4 T2 (EPUB path traversal fix) found that `_validate_epub_path()` in `app/backend/app/services/epub_service.py` used `Path()` without importing `pathlib.Path`. Every call from `scan_watch_folder()` was crashing with `NameError: name 'Path' is not defined`, which was silently swallowed by `scan_watch_folder()`'s broad `try/except Exception`.

**Impact:** **`scan_watch_folder()` has been silently failing since v1.2.0 merged.** The backend's EPUB auto-import on startup has been broken for days — every EPUB in `data/epubs/` was rejected.

**Why nobody noticed:** Silent try/except. The logs show a WARN per file (`[epub] Failed to auto-import %s: %s`) but without attention on the log level, nobody caught it.

**Fix:** WS4 T2 commit `d2da91b` adds `from pathlib import Path` to the imports of `epub_service.py` AND changes `import_epub()` to default `allowed_root` to `paths.EPUB_WATCH_DIR` (mandatory validation).

**Action items for next session:** **The user needs to manually rescan the EPUB watch folder after the v1.2.1 merge.** `data/epubs/` currently has 21 EPUBs. Depending on which ones were imported before the v1.2.0 regression and which were added after, some may be missing from the library DB. Either:
- Trigger a rescan via the frontend (after WS3 is done, there should be a "Scan folder" button)
- Call the backend endpoint directly: `POST /api/v1/epub/scan` or equivalent (check `routers/epub.py` for the exact URL)
- Run the backend with fresh logs and watch for `[epub] Auto-imported ...` messages
